# OSINT Agent — Client Lifestyle Enrichment

## La idea

Recopilar información pública del cliente (Instagram, LinkedIn, X/Twitter,
prensa) para enriquecer su perfil con gustos, hábitos y lifestyle. Esta
información sirve para dos cosas distintas:

1. **Investment alignment** — identificar oportunidades de inversión que
   conecten con sus intereses personales, más allá del mandato puro
2. **RM rapport** — dar al RM contexto personal para que sus conversaciones
   sean genuinas, no genéricas

Para el hackathon: los datos se simulan. La arquitectura es real.

## Caso de uso 1 — Investment alignment

El agente cruza el `lifestyleProfile` con el CIO list y el portfolio actual
para identificar oportunidades que conecten intereses personales con mandato.

### Lógica de matching

```python
LIFESTYLE_SECTOR_MAP = {
  "vino natural / biodinámico": ["Consumer Staples", "Agriculture", "LVMH"],
  "senderismo / naturaleza":    ["Outdoor & Sportswear", "Sustainable Tourism"],
  "rewilding / biodiversidad":  ["Forestry", "Biodiversity Bonds", "Impact Funds"],
  "agricultura regenerativa":   ["Agri-tech", "Sustainable Food"],
  "fly fishing":                ["Water quality", "Environmental services"]
}
```

### Ejemplo de output

```
LIFESTYLE-ALIGNED OPPORTUNITIES for Huber

  Pernod Ricard (FR0000120693)
  → Huber follows natural wine accounts and buys at farmer's markets
  → Pernod has a natural wine subsidiary (Kenwood Vineyards organic line)
  → CIO: HOLD — not a strong buy but a conversation opener
  → RM note: "podría interesarle el ángulo de vitivinicultura sostenible"

  Amer Sports (US-listed)
  → Huber usa Arc'teryx (owned by Amer Sports) en todas sus expediciones
  → CIO: BUY
  → RM note: "cliente usa la marca activamente — conexión personal fuerte"

  EcoTree (unlisted, alternative)
  → Inversión directa en proyectos de reforestación certificada
  → Conecta directamente con sus posts de rewilding y Atlantic Forest
  → RM note: "fuera del mandato standard pero encaja con su misión personal"
```

Estas oportunidades no reemplazan el swap resolver — son sugerencias
adicionales de conversación, marcadas como `type: lifestyle_opportunity`.

## Caso de uso 2 — RM rapport triggers

Enriquecer el `personalProfile.rapportTriggers` del DNA con datos OSINT
específicos y recientes que el RM puede usar en conversación.

### Antes (solo CRM)

```json
"rapportTriggers": [
  "Atlantic Forest project",
  "children's education"
]
```

### Después (CRM + OSINT)

```json
"rapportTriggers": [
  "Atlantic Forest project",
  "children's education",
  "recent hiking trip to Borneo — posted photos last month",
  "follows natural wine scene — @rawwine, @naturalwineclub",
  "co-signed palm oil letter — particularly vocal on this topic",
  "reads NZZ — referenced the regenerative agriculture article"
]
```

El message agent los usa exactamente igual que antes — la fuente
(CRM vs OSINT) no cambia el comportamiento, solo enriquece el pool.

## Integración en el pipeline

```
OSINT Agent (on demand o al cargar cliente)
    ↓
lifestyleProfile + rapportTriggers enriquecidos
    ↓
    ├── CRM Agent ← combina con DNA existente
    │       ↓
    │   ClientDNA completo (valores + lifestyle)
    │       ↓
    │   Message Agent (usa rapportTriggers enriquecidos)
    │
    └── Lifestyle Opportunity Matcher
            ↓
        oportunidades marcadas como lifestyle_opportunity
            ↓
        Dashboard — sección separada "Lifestyle Insights"
```

## Schema — nuevo campo en ClientDNA

```python
class LifestyleProfile(BaseModel):
    gastronomy: dict | None
    sports: dict | None
    lifestyle: dict | None
    professional: dict | None
    osint_sources: list[str]
    osint_confidence: float
    osint_retrieved_at: str

class ClientDNA(BaseModel):
    # ... campos existentes ...
    lifestyleProfile: LifestyleProfile | None = None
```

`None` si no hay datos OSINT disponibles — el sistema funciona igual sin él.

## Endpoint

```
POST /api/osint/enrich/{client_id}
Response: {
  "lifestyle_profile": LifestyleProfile,
  "new_rapport_triggers": list[str],
  "lifestyle_opportunities": list[Opportunity]
}

GET /api/osint/{client_id}
Response: LifestyleProfile ya guardado (si existe)
```

## Criterio de éxito

- El RM ve información personal relevante que no estaba en el CRM
- Al menos una lifestyle opportunity conecta interés personal con
  instrumento financiero real del CIO list
- Los rapport triggers OSINT se usan en el mensaje generado de forma
  natural (no todos a la vez, uno bien elegido)
- El sistema funciona igual si `lifestyleProfile` es null
